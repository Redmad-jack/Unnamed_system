# Frontend Guidelines

*Conscious Entity System — v0.1*

---

## 状态说明

v0.1 没有面向观众的前端界面。本文档当前主要定义**设计原则**和**待确认项**，为 v0.2 视觉层建设做铺垫。

---

## 1. 界面分层原则

系统有两套截然不同的界面，必须在视觉上明确区分：

| 界面 | 对象 | 目的 | 设计方向 |
|---|---|---|---|
| 访客展览界面 | 展览观众 | 沉浸式体验，感受实体的存在感 | 克制、有张力、非功能性 |
| 运营者监控面板 | 艺术家 / 研究者 | 清晰呈现内部状态，支持分析 | 功能性、信息密度高 |

这两套界面**不能使用相同的视觉语言**，绝对不能混用组件。

---

## 2. 视觉风格

**[ 待确认 ]** 整体视觉风格尚未最终确定。

以下为设计方向候选（供讨论，不是最终决定）：

**方向 A：极简主义 / 终端美学**
- 黑色背景，单色文字（白或浅灰）
- monospace 字体
- 无装饰性 UI 元素
- 文字是唯一的信息载体

**方向 B：有机不稳定感**
- 文字出现时有轻微的抖动或扰动效果
- 字间距、行间距随状态轻微变化
- 背景有低对比度的纹理或噪点

**方向 C：视觉层可替换**
- 核心输出层只提供文字 + 状态信号
- 视觉层是可插拔的"皮肤"，每次展览可以更换
- 内核稳定，外表多变

---

## 3. 色彩系统

**[ 待确认 ]** 具体色值尚未确定。

设计约束：
- 访客界面：低饱和度，避免使用高对比度的功能性颜色（红/绿）
- 运营者面板：可使用功能性颜色（状态指示、警告）
- 无障碍要求：文字对比度 ≥ 4.5:1（WCAG AA）

---

## 4. 字体

**[ 待确认 ]** 字体选型尚未确定。

约束：
- 必须支持中英文双语
- 访客界面优先考虑 monospace 或具有机械感的字体
- 运营者面板优先考虑可读性

---

## 5. 间距与布局

**[ 待确认 ]** 具体值待视觉设计确定后补充。

原则：
- 访客界面优先考虑全屏或极简布局，避免传统 UI 控件
- 运营者面板使用标准 dashboard 布局（侧边栏 + 主内容区）

---

## 6. 多通道输出设计原则

表达层输出有三个通道，必须作为独立可插拔层设计：

```
ExpressionOutput
  ├── text          → 文字通道（v0.1 实现）
  ├── spoken_text   → 声音通道（v0.2 实现）
  └── visual_mode   → 视觉状态通道（v0.2 实现）
```

**设计约束：**
- 新增通道不得修改 ExpressionEngine 的核心逻辑
- 每个通道有自己的渲染器，通道之间不互相依赖
- `visual_mode` 的取值集合必须在 ExpressionOutput 文档中明确列出，视觉层通过 switch/mapping 处理，不用 if-else 硬编码

**当前 `visual_mode` 取值：**
- `normal` — 正常显示
- `fragmented` — 文字片段化或间歇出现
- `disturbed` — 视觉扰动（具体效果 待确认）
- `silent` — 不输出文字，仅有视觉状态

---

## 7. 响应式原则

- v0.1（CLI）：不涉及
- v0.2+：访客界面设计以**全屏展览屏幕**为主要目标（通常 1920×1080 或更大）
- 运营者面板：支持普通笔记本屏幕（1440×900+）即可，移动端不作要求
- 不做移动端优先

---

## 8. 可拓展性要求

为未来展览的视觉多样性预留：
- 访客界面的视觉层应封装为独立模块，与业务逻辑解耦
- 状态到视觉的映射规则存储在 `config/expression_mappings.yaml`，不硬编码
- 添加新的 `visual_mode` 值只需修改 YAML 和渲染层，不改动后端逻辑
